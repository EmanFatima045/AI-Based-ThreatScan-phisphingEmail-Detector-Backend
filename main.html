System Online AI Model: Active Naive Bayes + Random Forest SQLite + FastAPI
01
Email Phishing
TF-IDF vectorization converts email text into numerical features. Naive Bayes calculates the probability that words like "urgent", "verify", "suspended" belong to a phishing email vs a legitimate one.
02
Malicious URLs
15+ engineered features: Shannon entropy of the domain, IP-based hostnames, suspicious keywords, HTTP vs HTTPS, subdomain count. Random Forest votes across 100 decision trees to classify.
03
Network Activity
Anomaly detection on IP addresses, request frequency, suspicious domains (e.g. .ru, .xyz, .tk TLDs), and traffic spikes. Flags DDoS patterns and data exfiltration attempts in real time.
04
Audit Logging
Every scan is stored in SQLite/PostgreSQL with timestamp, IP address, confidence score and risk level. Full audit trail for compliance. FastAPI auto-generates live Swagger docs at /docs.

// How It Works
The AI classifier uses TF-IDF vectorization to convert raw email text into numerical feature vectors, then Naive Bayes calculates the probability of phishing.
// URL Detection
Shannon entropy measures how random a domain name looks. Random Forest votes across 100 decision trees using 15 engineered URL features.
// Architecture
FastAPI backend with SQLAlchemy ORM, SQLite audit database, scikit-learn models, and this zero-dependency HTML frontend.
